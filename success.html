Security Features of Electron

  • nodeIntegration : false due to IPC
  • CSP rules - only get content from trusted sources, block inline JS thus stop Injection attacks, enable reporting

Pros of Electron

  • NodeJS runtime - exposes a wide set of libaries and capabilities of Node,npm init
  • Node Integration even on the Client side
  • Ability to use common web technologies - HTML, jQuery etc, Dev team is familiar with
  • Cross Platform, Open Source
  • Integration with CI systems like Jenkins
  • Electron is more popular than .NET in terms of GitHub stars,forks
  • Secure Coding practices: https://github.com/electron/electron/blob/master/docs/tutorial/security.md

Cons of Electron

  • Electron App comes with its own version of Chromium which is quite heavy
  • Native Dependencies would make the app not cross-platform
  • No source code encryption or hidden - we would need to package source code with asar + Obfuscation by some library
